2026-06-10 — PHASE 1 PRE-REGISTRATION (pre-registered via pm-update, 2026-06-10; Brett). Verbatim:
1. Slice: past tense only (v1-equivalent) as primary; all-tense pooled mean as a sensitivity arm. Native-speaker filter as you proposed — endorsed, and report the (trivial) effect.
2. Normalization: participant-level z-scores (each rater standardized over their own responses), cell value = mean z across that cell's raters. Rationale: median five raters per cell makes rater scale-use the dominant nuisance, and per-rater standardization is the cheap, standard correction — we're not reproducing W&R's ordinal-model normalization, just citing it once read.
3. Primary threshold: z̄ > 0. The raw distribution being U-shaped means any mid-scale cut lands in the density trough, so the choice is less consequential than it looks — but z̄ > 0 specifically reads as "above this rater pool's own average," and its 0.356 base rate keeps frame columns furthest from degeneracy, which the BIC node fits need.
4. Sensitivity grid: z̄ > 0.25; raw mean ≥ 4; raw mean ≥ 4.5. Pre-registered robustness criterion: the verdict — sign of cv.diff_mean and the eigen-dominance band — must be invariant across the grid. If it isn't, threshold-dependence is itself the reported result; no arm gets promoted post hoc.
5. Continuous check: eigen spectrum of the undichotomized z̄ matrix reported alongside, satisfying PLAN §3's ordinal clause without new modeling code.
6. Missing cells: exclude verbs with any missing cell in the analyzed slice; report how many verbs that costs (bounded above by 264). Flag any frame with post-dichotomization base rate outside [0.02, 0.98]; they stay in, flagged.

Calibration note (pm, not a decision): with 1,007 verbs from a single putative category, the expected boring outcome is a strong first dimension (general embeddability); eigen-dominance alone will not adjudicate the theories on this dataset (it will on the Phase 3 cross-category grid). Informative quantities here: the CV comparison, residual structure after the first dimension, and which frames carry projectibility. High dominance is not a verdict for the reflective view.

2026-06-10 — Phase 3 sampling frame, pre-registered via pm-update, 2026-06-10. Verbatim:

F1 — Strata and sizes (target ≈ 450 lexemes). (a) Adjectives, n = 220: sampled from lemmas POS-tagged adjective in the frequency frame (F2), stratified into four log-frequency bands by rank within tag — 1–500, 501–2,000, 2,001–10,000, 10,001–50,000 — 55 per band, simple random within band. (b) Prepositions: the full inventory of single-word prepositions as listed in CGEL ch. 7 — executor extracts the list from the chapter itself (source-grounded; expected order ~100; log the exact count and provenance). (c) Determinatives: the determinative list from Reynolds 2021's 138-item matrix, reused verbatim — already source-grounded and machine-readable. (d) Boundary set, n = 16: the forced-inclusion list in diagnostics.yaml, analyzed with and without, per RQ3. (e) Category-blind stratum, n = 60: random lemmas from the top 10,000 by frequency, drawn with no POS conditioning. This stratum is the structural answer to Croft's opportunism charge at the sampling stage: if cluster structure emerges among items selected with no category labels at all, the categories aren't artefacts of the seed lists.

F2 — Frequency frame. Primary: Kilgarriff's lemmatized BNC frequency list (free, POS-tagged, stable artifact); executor verifies availability, logs URL + file hash to the manifest before sampling. Fallback if unavailable or truncated: a stated open corpus tagged with a stated tagger version, logged the same way. Either way the frame file itself goes in data/ so the sample is reproducible from the repo alone.

F3 — Circularity firewall. Frame POS labels (and CGEL list membership) are used only to define strata for coverage and for known-groups validity checks. No analysis conditions on them as ground truth; the measurement models see an unlabeled lexeme × item matrix. This goes in PLAN §3 as an amendment and in the paper's methods verbatim-in-spirit.

F4 — Sampling mechanics. Seed = 26 for all draws. Sampling happens in one script, one commit, before any sentence is instantiated. Lexemes appearing in multiple strata (a frequent preposition also in the blind stratum) are deduplicated toward the more specific stratum, with the collision count logged.

F5 — Battery gate. Instantiation is blocked until TO_VERIFY item B clears: every source: field in diagnostics.yaml verified against the cited text. Item floor: ≥40 items per lexeme — current arithmetic is ~25 judgment items + ~18 corpus operationalizations ≈ 43, satisfying the power floor conditional on the battery surviving verification; if pruning drops it below 40, battery expansion comes back to the pm node before anything runs. Unconstructable lexeme × frame instantiations go in the template-exception table, never silent hand-edits.

F6 — Gold sample (DSL/PPI anchor). Simple random sample of 600 cells (uniform inclusion probability, seed = 26), drawn after the grid is instantiated and before any LLM rating exists — independence of gold from judge output is what makes the correction valid. Coding: me-the-human blind to judge ratings; a 200-cell double-coded subset for κ (Pullum is the candidate second coder — that's an email for you, not the executor). Published-judgment agreement (CGEL stars; Sprouse–Almeida hard region) remains judge validation, firewalled from the correction estimate, as pre-registered in the chat record.

F7 — Analysis mirror. Phase 3 runs the Phase 1 pipeline unchanged — same arms, same amended dominance bands (>3 / <3), bootstrap T2 procedure as now implemented — on the full grid and on per-stratum subsets, plus the RQ3 misfit ranking with the boundary set in/out. No new analytic degrees of freedom without a pm-update.
